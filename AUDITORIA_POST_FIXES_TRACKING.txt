Hallazgos altos

1. Permisos de grupos/miembros siguen dependiendo de tablas directas.
   Severidad: Alta.
   Estado: Pendiente.
   Archivo y lineas:
   - src/services/groups.service.js:125
   - src/services/groups.service.js:157
   - src/services/groups.service.js:177
   - src/services/groups.service.js:254
   Evidencia:
   createGroup, updateGroup, deleteGroup, addMember y removeMember operan directo sobre groups/group_members.
   La migracion nueva solo habilita RLS para jobs y group_join_requests en supabase/20260618_authorization_fixes.sql:56 y 208.
   Pasos para reproducir:
   Intentar insertar/eliminar group_members directo desde un cliente autenticado si la DB no tiene policies manuales adicionales.
   Impacto:
   Miembros comunes podrian administrar grupos/miembros si la base no tiene RLS externa.
   Correccion minima recomendada:
   Migracion RLS/RPC para groups y group_members.
   Confianza: Alta.

2. Update RLS de jobs permite demasiadas columnas para edicion por grupo.
   Severidad: Alta.
   Estado: Pendiente.
   Archivo y lineas:
   - supabase/20260618_authorization_fixes.sql:120
   Evidencia:
   La policy permite update si editable_by_group y membresia; no limita columnas como user_id, group_id, editable_by_group.
   Pasos para reproducir:
   Usuario miembro actualiza directamente un job editable por grupo cambiando campos sensibles.
   Impacto:
   Alteracion de propiedad/integridad del trabajo.
   Correccion minima recomendada:
   Mover edicion a RPC validada o restringir privilegios de columnas/triggers.
   Confianza: Media-Alta.

Riesgos de integridad de datos

1. Edicion por grupo de jobs puede modificar columnas sensibles si se llama directamente a Supabase.
2. Gestion de groups/group_members no tiene barrera server-side visible en repo.
3. El tooling roto impide validar automaticamente tests agregados.
4. Estados async en MonthlyPanelPage y useJobFormData pueden actualizar tras desmontaje.

Orden recomendado para los proximos 3 fixes

1. RLS/RPC faltante para groups y group_members.
2. Endurecer update de jobs con RPC o restricciones por columna.
3. Reparar tooling minimo de lint/tests.
